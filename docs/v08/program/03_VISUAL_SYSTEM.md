# 03 — Visual and Aesthetic System

## Direction: archaeological modernism

Preserve apparatus topology and causal relations without simulating obsolete hardware.

Use:

- stable channels and lanes;
- compact status lamps;
- signal-path alignment;
- visible selection;
- tactile borders/inset editors;
- high-contrast focus;
- semantic motion;
- distinct apparatus and poetic registers.

Reject:

- fake screws, platters, metal, wood, tubes;
- decorative waveforms;
- knobs for values better shown numerically;
- low-contrast neumorphism;
- arbitrary DAW rainbow colors;
- generic SaaS cards.

## Apparatus register

- dark field;
- monospaced UI;
- compact labels;
- clear borders;
- tabular numerals;
- semantic accents.

## Poetic register

- more air;
- line integrity;
- distinct reading face if appropriate;
- reduced chrome;
- readable language never replaced by abstract bars.

## Semantic colors

Define tokens for:

- Cue
- Surface
- active
- warning
- danger
- stale
- muted
- focus

Every color state also has text/icon/shape.

## Density

Three levels:

1. summary row;
2. collection;
3. selected inspector.

Do not render all fields at once.

## Motion

Motion only for selection, insertion, queue/launch, consumed event, saved/muted state. Respect reduced motion.

## Storybook state catalogue

Every component:

- default;
- hover;
- focus-visible;
- selected;
- disabled;
- invalid;
- empty;
- loading;
- long text;
- narrow width;
- dense;
- reduced motion.

Composite stories include 33 banks, 270 tokens, long IDs/names, route with many slots, deep pattern, long trigger, 375 portrait, 844×390 landscape.
